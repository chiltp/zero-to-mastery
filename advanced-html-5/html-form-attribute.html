HTML Form Attributes

The Action Attribute

The action attribute essentially determines where the form data goes after submission, allowing you to send it to a specific URL or script for further processing.

In this example, the action attribute is set to "https://www.example.com/submit-form", indicating that when the form is submitted, the data will be sent to that URL. The method attribute specifies the HTTP method to be used (POST in this case).

Note: If the action attribute is omitted, the action is set to the current page.

The Method Attribute

The method determines how the form data is delivered to the serverâ€”whether it's appended to the URL using the "GET" method or sent separately using the "POST" method.

1. GET method

When you use the GET method, the form data is appended to the URL in the form of query parameters. It's like writing the package contents on the outside of the package. This method is commonly used when you want to retrieve or search for data from the server.

In this example, a search form is created. When the user submits the form, the form data (the query entered by the user) will be appended to the URL as query parameters. For instance, if the user searches for "apple," the URL will become: https://www.example.com/search?query=apple. This allows the server to process the query parameter and return relevant search results.

2. POST method

When you use the POST method, the form data is sent separately from the URL, like an enclosed package. This method is commonly used when you want to send sensitive information or when the form data can modify or update data on the server.

In this example, a login form is created. When the user submits the form, the form data (the entered username and password) will be sent separately from the URL. The data is not visible in the URL, which is important for sensitive information like passwords. The server can then handle the submitted data securely and perform authentication or authorization processes.

Tip: Always use POST if the form data contains sensitive or personal information!
